# 경영 관리

## 개요
이 문서는 구성관리·변경관리·장애관리에서 쌓인 데이터를 경영진 관점에서 요약·보고하는 자료를 관리하는 기준을 안내합니다.

## 목적
- 실무 데이터(자산/변경/장애)를 경영진이 빠르게 이해할 수 있는 형태로 가공하여, 의사결정에 필요한 현황을 정기적으로 전달합니다.

## 하위 폴더
| 폴더명 | 설명 |
|---|---|
| `_업무보고/` | 경영진 보고용 브리핑·리포트 산출물(PDF 등)을 보관 |
| `구축산출물/` | 개발/유지보수 담당자용 공식 기술 문서(테스트결과서·프로그램명세서·데이터베이스명세서·아키텍처결과서·표준정의서, PDF). 카탈로그는 [구축산출물목록.md](구축산출물목록.md) |

## 보고 프로세스
- (작성 예정) 데이터 취합 → 요약본 작성 → 검토 → 보고 → 배포 단계별 절차

## 참고 및 관리 방법
- 보고서 용어 수준은 대상에 맞춥니다: 비개발자 임원용은 기술 용어(PK/FK/테이블 등) 없이 업무 용어로, 이사진 대상 ERD·아키텍처 보고처럼 기술적 정확성이 중요한 경우엔 물리명/논리명·PK/FK 등을 명시적으로 표기합니다.
- 보고서 산출물은 `_업무보고/` 폴더에 `YYYYMMDD_제목.pdf` 형식으로 저장하고, 위 "보고 이력 목록" 표와 [업무보고목록.md](업무보고목록.md)를 함께 갱신합니다.
- (작성 예정) 보고 주기, 배포 대상, 버전 관리 기준
