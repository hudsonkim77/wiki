# 백업 관리

## 개요
이 문서는 CI(시스템) 데이터의 정기 백업과 유사시 복구(소산 관리 포함)를 관리하는 기준을 안내합니다. `RAW/erd-참고용.png`의 **BCK(백업관리)** 항목에 대응합니다.

## 목적
- 정기 백업 수행 여부와 결과를 추적합니다.
- 장애/재해 발생 시 어떤 백업으로부터 복구했는지 추적합니다(양방향: 백업 ↔ 복구).

## 백업/복구 프로세스
- (작성 예정) 정기 백업 수행 → 보관 → (필요 시) 복구 실행 → 복구 검증 단계별 절차

## 필드 정의 (BACKUP 테이블)
백업(정기)과 복구(유사시) 모두 이 테이블에 `ACTION_TYPE`으로 구분해 기록합니다.

| 필드명 | 설명 | 비고 |
|---|---|---|
| BACKUP_ID | 백업/복구 고유 ID | **PK**. 예: `BCK_20260711_001` |
| ACTION_TYPE | 구분 | 백업/복구 |
| BACKUP_DT | 수행 일시 | |
| STATUS | 결과 | 성공/실패 |
| RETENTION_UNTIL_DT | 보관 만료일 | ACTION_TYPE=백업인 경우만 |
| RESTORED_FROM_BACKUP_ID | 복구에 사용한 백업 ID | FK → 이 테이블 자기참조(ACTION_TYPE=복구인 경우만) |
| OWNER_TEAM | 담당 부서 | |

## 구성관리와의 연결
- 백업/복구 1건은 여러 CI를 한 번에 포괄할 수 있고(예: "전산실 백업"이 서버 다수 포함), CI 1건도 여러 백업 이력을 가질 수 있어 **N:M** 관계입니다.
- 실제 연결은 **`BACKUP_CI_MAP` 중간 테이블**을 통해 관리하며, 데이터 파일은 [백업이력/](백업이력/README.md) 폴더에 둡니다.
- 참고 그림의 "BCK ↔ CFG (데이터/복구)"는 양방향 관계로, 정기 백업과 유사시 복구(소산 관리 포함)를 모두 포함합니다.

## 생성된 테이블 파일
| 파일 위치 | 설명 | 건수 |
|---|---|---|
| `BACKUP.csv` (이 폴더) | 위 필드 정의를 구현한 백업/복구 마스터 테이블 | 1건(`BAK_20260712_001`, 가상 예시 데이터) |
| [`백업이력/BACKUP_CI_MAP.csv`](백업이력/README.md) | BACKUP-CI 중간(연결) 테이블 | 0건(스키마만 우선 반영) |
