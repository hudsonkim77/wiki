# 형상 관리 (소프트웨어 형상관리)

## 개요
이 문서는 소스코드·빌드산출물·문서 등의 **버전/베이스라인**을 관리하는 기준을 안내합니다. `RAW/erd-참고용.png`의 **SCM(형상관리)** 항목에 대응합니다.

> **[3_구성관리](../3_구성관리/README.md)와의 구분**: 3_구성관리(CMDB)는 서버·스토리지·네트워크 등 "인프라 자산이 무엇이고 서로 어떻게 연결되어 있는지"를 관리합니다. 이 폴더(형상관리)는 그와 별개로 "소프트웨어/문서 산출물의 특정 시점 버전(베이스라인)이 무엇인지"를 관리합니다 — 두 도메인은 이름은 비슷하지만 관리 대상이 다릅니다.

## 목적
- 배포 가능한 버전(베이스라인)을 명확히 정의하고, 어떤 CI(시스템)에 적용되는 버전인지 추적합니다.
- 배포관리([7_배포관리](../7_배포관리/README.md))에서 어떤 베이스라인이 배포되는지 참조할 수 있게 합니다.
- 문제관리(PRB)의 근본원인 해결(영구조치) 결과가 새 베이스라인으로 이어지는 흐름을 반영합니다.

## 형상 관리 프로세스
- (작성 예정) 베이스라인 후보 확정 → 검토·승인 → 베이스라인 확정(태깅) → 배포/연계 참조 단계별 절차

## 필드 정의 (BASELINE 테이블)

| 필드명 | 설명 | 비고 |
|---|---|---|
| BASELINE_ID | 베이스라인 고유 ID | **PK**. 예: `BSL_20260711_001` |
| BASELINE_NAME | 베이스라인명 | |
| VERSION | 버전 표기 | 예: `v1.2.0` |
| BASELINE_TYPE | 유형 | 소스코드/빌드산출물/문서 |
| STATUS | 진행 상태 | 작성중/승인/배포됨/보관 |
| CREATED_DT | 생성 일시 | |
| APPROVED_DT | 승인 일시 | |
| OWNER_TEAM | 담당 부서 | |
| RESOLVED_PROBLEM_ID | 이 베이스라인의 근거가 된 문제 ID | FK → [6_문제관리](../6_문제관리/README.md) PROBLEM.PROBLEM_ID (선택) |

## 구성관리와의 연결
- 베이스라인 1건은 여러 CI(시스템)에 적용될 수 있고, CI 1건도 여러 베이스라인 이력을 가질 수 있어 **N:M** 관계입니다.
- 실제 연결은 **`BASELINE_CI_MAP` 중간 테이블**을 통해 관리하며, 데이터 파일은 [형상이력/](형상이력/README.md) 폴더에 둡니다.

## 문제관리와의 연결
- 참고 그림의 "PRB → SCM (Root Cause Analysis → Code & Document Versioning)" 흐름대로, 베이스라인 1건의 근거 문제는 최대 1건이라 **N:1** 관계이며 `RESOLVED_PROBLEM_ID` FK로 직접 연결합니다.

## 배포관리·연계관리와의 관계
- 어떤 베이스라인이 실제로 배포되었는지는 [7_배포관리](../7_배포관리/README.md)의 `DEPLOY.csv`에서 참조합니다(별도 FK 없이 DESC에 버전 기재 권장).
- [14_연계관리](../14_연계관리/README.md) INTERFACE.csv가 `BASELINE_ID`로 이 테이블을 참조합니다(참고 그림 "SCM → INT").

## 생성된 테이블 파일
| 파일 위치 | 설명 | 건수 |
|---|---|---|
| `BASELINE.csv` (이 폴더) | 위 필드 정의를 구현한 베이스라인 마스터 테이블 | 0건(스키마만 우선 반영) |
| [`형상이력/BASELINE_CI_MAP.csv`](형상이력/README.md) | BASELINE-CI 중간(연결) 테이블 | 0건(스키마만 우선 반영) |
