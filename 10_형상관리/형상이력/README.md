# 형상이력 (BASELINE_CI_MAP)

## 개요
이 폴더는 형상관리(`BASELINE`)와 구성관리(`CI`) 사이의 **N:M 관계를 연결하는 중간(연결) 테이블**을 보관합니다.
상위 도메인 설명은 [10_형상관리](../README.md), 구성관리 설명은 [3_구성관리](../../3_구성관리/README.md)를 참고하세요.

## 파일 목록
| 파일명 | 설명 | 건수 |
|---|---|---|
| BASELINE_CI_MAP.csv | 어떤 베이스라인이 어떤 CI(시스템)에 적용되는지 매핑 | 312건(전체 CI가 `BSL_20260712_002` 초기 베이스라인 v1.0에 편입, 2026-07-12) |

## 필드 정의
| 필드명 | 설명 | 비고 |
|---|---|---|
| MAP_ID | 매핑 고유 ID | **PK** |
| BASELINE_ID | 베이스라인 ID | **FK → ../BASELINE.csv** |
| BASELINE_VERSION | 연결 시점의 베이스라인 버전 스냅샷 | `../BASELINE.csv`의 `VERSION`을 편입 시점에 복사해 둔 값(2026-07-12 추가). BASELINE 원본이 이후 폐기/개정되어도 "이 CI가 그때 어떤 버전에 속했는지"가 이 값으로 고정 보존됨 |
| CI_ID | 적용 대상 구성항목 ID | **FK → ../../3_구성관리/CI.csv** |
| BASELINE_AREA | 베이스라인 영역 구분 | `인력` / `응용` / `인프라` 중 하나(2026-07-12 추가). CI.CI_TYPE 기준 자동 분류(HUMAN→인력, WEB_PAGE→응용, 그 외→인프라) |
| APPLIED_DT | 이 CI가 베이스라인에 편입된 일시 | 2026-07-12 추가 |
| NOTE | 연결 근거 메모 | |

### 스키마 설계 근거(2026-07-12)
BASELINE 마스터에 이미 `VERSION` 컬럼이 있어 `BASELINE_ID`만으로도 조인해서 버전을 알 수 있지만(정규화 관점에서는 이걸로 충분), 아래 두 이유로 `BASELINE_VERSION`을 매핑 테이블에 스냅샷으로 추가 보존한다.
1. **불변성 보장**: 베이스라인이 이후 재승인·개정되어 `VERSION`이 바뀌더라도, 과거 시점에 이 CI가 속했던 버전 값이 매핑 테이블에 고정되어 감사(audit) 추적이 가능하다.
2. **조회 성능/단순성**: 어떤 CI가 어느 버전에 속하는지 확인할 때 BASELINE.csv까지 조인하지 않고 이 표만으로 바로 확인 가능하다.

`BASELINE_AREA`는 "통합 초기 베이스라인 정의서 v1.0"(인력/응용/인프라 3개 영역 구분)의 요구사항을 데이터로 반영하기 위해 추가했다.

## 참고 및 관리 방법
- 베이스라인 확정 시 [`../BASELINE.csv`](../BASELINE.csv)에 마스터 행을 먼저 추가하고, 적용 대상 CI를 이 표에 (BASELINE_ID, CI_ID) 쌍으로 행 단위 추가합니다.
