# 운영상태 관리

## 개요
이 문서는 CI(시스템)의 실시간 운영 상태를 점검·모니터링하는 기준을 안내합니다. `RAW/erd-참고용.png`의 **OPS(운영상태관리)** 항목에 대응합니다.

## 목적
- 시스템 모니터링을 통해 장애를 사전에 예방합니다.
- 모니터링 중 이상 신호는 이벤트([12_이벤트관리](../12_이벤트관리/README.md))로 기록해 필요 시 장애로 확대합니다.
- 예방 활동의 결과는 서비스수준관리(SLA)의 실적 측정 근거로 활용됩니다(참고 그림의 "운영상태를 통해 장애를 예방하고 이에 따른 서비스 수준관리가 가능" 흐름).

## 운영 점검 프로세스
- (작성 예정) 점검 대상 CI 정의 → 주기적 상태 점검 → 이상 감지 시 이벤트 생성 → SLA 반영 단계별 절차

## 필드 정의 (OPS_STATUS 테이블)

| 필드명 | 설명 | 비고 |
|---|---|---|
| OPS_ID | 운영상태 점검 고유 ID | **PK**. 예: `OPS_20260711_001` |
| CHECK_DT | 점검 일시 | |
| STATUS | 점검 결과 | 정상/주의/장애 |
| METRIC_SUMMARY | 점검 지표 요약 | |
| NOTE | 비고 | |

## 구성관리와의 연결
- 점검 1건은 여러 CI(복합 서비스 단위 점검)를 포괄할 수 있고, CI 1건도 여러 점검 이력을 가질 수 있어 **N:M** 관계입니다.
- 실제 연결은 **`OPS_CI_MAP` 중간 테이블**을 통해 관리하며, 데이터 파일은 [운영이력/](운영이력/README.md) 폴더에 둡니다.

## 다른 도메인과의 연결
- 이 도메인의 점검 결과가 이상 신호를 만들면 [12_이벤트관리](../12_이벤트관리/README.md) EVENT.csv의 `SOURCE_OPS_ID`로 연결됩니다.
- 이 도메인의 예방 활동 실적은 [9_서비스수준관리](../9_서비스수준관리/README.md)의 `SLA_OPS_MAP` 중간 테이블로 연결됩니다.

## 생성된 테이블 파일
| 파일 위치 | 설명 | 건수 |
|---|---|---|
| `OPS_STATUS.csv` (이 폴더) | 위 필드 정의를 구현한 운영상태 점검 마스터 테이블 | 0건(스키마만 우선 반영) |
| [`운영이력/OPS_CI_MAP.csv`](운영이력/README.md) | OPS_STATUS-CI 중간(연결) 테이블 | 0건(스키마만 우선 반영) |
