# 변경히스토리 (CHANGE_CI_MAP)

## 개요
이 폴더는 변경관리(`CHANGE`)와 구성관리(`CI`) 사이의 **N:M 관계를 연결하는 중간(연결) 테이블**을 보관합니다.
설계 근거와 ERD는 [3_구성관리](../../3_구성관리/README.md), 상위 도메인 설명은 [1_변경관리](../README.md)를 참고하세요.

## 파일 목록
| 파일명 | 설명 | 건수 |
|---|---|---|
| CHANGE_CI_MAP.csv | 어떤 CI가 어떤 변경 티켓에 연결되어 있는지 매핑. `../../RAW/CFG.csv`에서 추출한 60건 + [5_ITSM관리](../../5_ITSM관리/README.md) 통합관리대시보드 관련 5건 | 65건 |

## 필드 정의
| 필드명 | 설명 | 비고 |
|---|---|---|
| MAP_ID | 매핑 고유 ID | **PK**. `MAP_0001` ~ `MAP_0065` |
| CHG_TICKET_ID | 변경 티켓 ID | **FK → ../CHANGE.csv** |
| CI_ID | 영향받은 구성항목 ID | **FK → ../../3_구성관리/CI.csv** |
| APPLIED_DT | 해당 CI에 변경이 적용된 일시 | 원본에 없어 현재 공란, 추후 보완 필요 |
| WORK_NOTE | 해당 CI 기준 작업 메모 | 원본에 없어 현재 공란, 추후 보완 필요 |

## 참고 및 관리 방법
- 새로운 변경 적용 시 이 표에 (변경 티켓, 영향받은 CI) 쌍을 행 단위로 추가합니다.
