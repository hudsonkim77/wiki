# 서비스 수준 관리 (SLA)

## 개요
이 문서는 서비스별 목표 수준(가용성, 응답시간 등)을 정의하고 실측치와 비교·관리하는 기준을 안내합니다. `RAW/erd-참고용.png`의 **SLA(서비스수준관리)** 항목에 대응하며, 참고 그림상 전체 흐름의 최종 성과 검토(Review) 직전 단계입니다.

## 목적
- 서비스 품질을 수치로 정의하고 목표 대비 실적을 추적합니다.
- 목표 미달(SLA breach) 발생 시 원인이 된 장애를 함께 추적합니다.
- 운영상태관리(OPS)의 예방 활동과 배포관리(DEP)의 배포 결과를 실적 측정 근거로 반영합니다.

## SLA 관리 프로세스
- (작성 예정) 목표 정의 → 주기별 측정 → 목표 대비 평가 → 미달 시 원인 분석 단계별 절차

## 필드 정의 (SLA 테이블)

| 필드명 | 설명 | 비고 |
|---|---|---|
| SLA_ID | SLA 항목 고유 ID | **PK**. 예: `SLA_20260711_001` |
| SERVICE_NAME | 대상 서비스명 | |
| METRIC_TYPE | 측정 지표 | 가용성%/응답시간/처리시간 |
| TARGET_VALUE | 목표치 | |
| PERIOD | 측정 기간 | 예: `2026-07` |
| MEASURED_VALUE | 실측치 | |
| STATUS | 평가 결과 | 충족/미달 |
| OWNER_TEAM | 담당 부서 | |
| DEPLOY_ID | 측정 대상 배포 ID | FK → [7_배포관리](../7_배포관리/README.md) DEPLOY.DEPLOY_ID (선택) |

## 장애관리와의 연결
- SLA 미달 1건은 여러 장애가 원인일 수 있고, 장애 1건도 여러 SLA 항목에 영향을 줄 수 있어 **N:M** 관계입니다. (참고 그림: "장애관리를 통해 운영상태·이벤트 알람 대응 품질을 측정")
- 실제 연결은 **`SLA_INCIDENT_MAP` 중간 테이블**을 통해 관리하며, 데이터 파일은 [SLA이력/](SLA이력/README.md) 폴더에 둡니다.

## 운영상태관리와의 연결
- 참고 그림의 "운영상태를 통해 장애를 예방하고 이에 따른 서비스 수준관리가 가능"한 주요 흐름을 반영합니다. SLA 1건은 여러 운영상태 점검 결과를 근거로 삼을 수 있고, 점검 1건도 여러 SLA 항목의 근거가 될 수 있어 **N:M** 관계입니다.
- 실제 연결은 **`SLA_OPS_MAP` 중간 테이블**을 통해 관리하며, 데이터 파일은 [SLA이력/](SLA이력/README.md) 폴더에 둡니다.

## 배포관리와의 연결
- SLA 1건은 특정 배포 건의 서비스 품질을 모니터링하는 경우가 있어(참고 그림 "DEP → SLA Monitoring"), 배포 1건에 여러 SLA 측정치가 달릴 수 있는 **N:1** 관계로 보고 `DEPLOY_ID` FK로 직접 연결합니다.
