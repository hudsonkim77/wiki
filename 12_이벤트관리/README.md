# 이벤트 관리

## 개요
이 문서는 CI에서 발생하는 모니터링 이벤트(경고/정보/긴급)를 기록하고, 필요 시 장애로 확대(에스컬레이션)하는 기준을 안내합니다. `RAW/erd-참고용.png`의 **EVT(이벤트관리)** 항목에 대응합니다.

## 목적
- 장애로 이어지기 전 단계의 신호(이벤트)를 놓치지 않고 기록합니다.
- 이벤트가 실제 장애로 확대된 경우 해당 장애와 연결해 추적합니다.
- 운영상태관리(OPS)의 모니터링 결과로 생성된 알림/로그를 이벤트로 기록합니다.

## 이벤트 처리 프로세스
- (작성 예정) 이벤트 수집 → 필터링/분류 → 확인 → (필요 시) 장애 에스컬레이션 → 종결 단계별 절차

## 필드 정의 (EVENT 테이블)

| 필드명 | 설명 | 비고 |
|---|---|---|
| EVENT_ID | 이벤트 고유 ID | **PK**. 예: `EVT_20260711_001` |
| EVENT_TITLE | 이벤트 제목 | |
| EVENT_TYPE | 이벤트 유형 | 경고/정보/긴급 |
| CI_ID | 발생 구성항목 ID | FK → [3_구성관리](../3_구성관리/README.md) CI.CI_ID |
| SOURCE_OPS_ID | 이 이벤트를 생성한 운영상태 점검 ID | FK → [11_운영상태관리](../11_운영상태관리/README.md) OPS_STATUS.OPS_ID (선택) |
| DETECTED_DT | 감지 일시 | |
| EVENT_STATUS | 진행 상태 | 신규/확인/종결 |
| ESCALATED_INCIDENT_ID | 확대된 장애 ID | FK → [2_장애관리](../2_장애관리/README.md) INCIDENT.INCIDENT_ID (선택, 장애로 확대된 경우만) |

## 다른 도메인과의 연결
- 이벤트 1건의 발생 CI·생성 주체(운영상태 점검)·확대된 장애는 각각 최대 1건이라 **N:1** 관계이며, **중간 테이블 없이 FK로 직접 연결**합니다. ([2_장애관리](../2_장애관리/README.md) INCIDENT.csv의 `CAUSED_BY_CHG_TICKET_ID`와 동일한 패턴)
- 참고 그림의 "OPS → EVT (알림 & 로그)" 흐름을 `SOURCE_OPS_ID`로 반영했습니다.

## 생성된 테이블 파일
| 파일 위치 | 설명 | 건수 |
|---|---|---|
| `EVENT.csv` (이 폴더) | 위 필드 정의를 구현한 이벤트 마스터 테이블 | 0건(스키마만 우선 반영) |
