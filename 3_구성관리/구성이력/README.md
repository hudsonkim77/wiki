# 구성이력 (CI_HISTORY)

## 개요
이 폴더는 구성관리(`CI`) 자산의 등록/폐기(추가/삭제) 이력을 기록하는 로그 테이블을 보관합니다.
CI 자체의 현재 상태는 [3_구성관리/CI.csv](../CI.csv)를 참고하고, 이 폴더는 "언제, 무엇이, 왜 추가/삭제되었는지"의 이력만 담당합니다.

## 파일 목록
| 파일명 | 설명 | 건수 |
|---|---|---|
| CI_HISTORY.csv | CI 추가/삭제 이력 마스터 | 4건(최초 백필) + 이후 [5_ITSM관리](../../5_ITSM관리/README.md) `pages/3_구성관리.py` CI 등록/삭제 화면을 통한 실사용 건 누적 |

## 필드 정의
| 필드명 | 설명 | 비고 |
|---|---|---|
| HISTORY_ID | 이력 고유 ID | **PK**. `HIST_0001`부터 순차 증가 |
| CI_ID | 대상 CI ID | **FK(느슨한 참조) → ../CI.csv.CI_ID** — 삭제된 CI는 CI.csv에 더 이상 없을 수 있어 강한 FK로 걸지 않음 |
| CI_NAME | 대상 CI 명칭(스냅샷) | 삭제 후에도 이력에서 알아볼 수 있도록 CI.csv 값을 복사해 저장 |
| CI_CATEGORY | 대상 CI 카테고리(스냅샷) | 위와 동일한 이유로 복사 저장 |
| ACTION | 조치 구분 | `ADDED`(추가) / `REMOVED`(삭제) |
| ACTION_DT | 조치 일시 | |
| ACTION_BY | 조치자 ID | |
| NOTE | 비고 | 근거(변경 티켓 등) 또는 조치 경위 |

## 대시보드 연동
[5_ITSM관리](../../5_ITSM관리/README.md) `홈.py`의 "현황" 스코어보드 "구성관리(추가)" / "구성관리(삭제)" 지표는 이 파일의 `ACTION` 값 개수를 집계해 표시합니다(`CHG_20260711_009` 이전에는 이력 로그가 없어 각각 4/0으로 고정 표기했었음).

## 참고 및 관리 방법
- CI 추가/삭제 시 반드시 이 파일에 이력 행을 남깁니다. `pages/3_구성관리.py`의 CI 등록/삭제 화면을 사용하면 자동으로 기록됩니다.
- 최초 4건(`HIST_0001`~`HIST_0004`)은 [5_ITSM관리](../../5_ITSM관리/README.md) 통합관리대시보드 최초 구축(`CHG_20260711_001`) 당시 등록된 CI 4건(`CFG_WEB_019`~`022`)을 소급 기록한 것입니다.
