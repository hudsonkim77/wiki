# 테스트 관리

## 개요
이 문서는 배포 전/후 검증·확인 테스트를 기록하는 기준을 안내합니다. `RAW/erd-참고용.png`의 **TST(테스트관리)** 항목에 대응합니다.

## 목적
- 배포 건별로 어떤 검증이 이뤄졌고 결과가 어땠는지 추적합니다.
- 테스트 결과를 연계관리(INT)의 외부 연동 검증 근거로 활용합니다.

## 테스트 처리 프로세스
- (작성 예정) 테스트 케이스 설계 → 실행 → 결과 판정(PASS/FAIL) → (필요 시) 재테스트 단계별 절차

## 필드 정의 (TEST_CASE 테이블)
[7_배포관리](../7_배포관리/README.md)의 DEPLOY 테이블과 연결되는 별도 테이블로 설계합니다.

| 필드명 | 설명 | 비고 |
|---|---|---|
| TEST_ID | 테스트 고유 ID | **PK**. 예: `TST_20260711_001` |
| DEPLOY_ID | 검증 대상 배포 ID | FK → [7_배포관리](../7_배포관리/README.md) DEPLOY.DEPLOY_ID |
| TEST_TYPE | 테스트 유형 | 기능테스트/회귀테스트/성능테스트 |
| TEST_RESULT | 테스트 결과 | PASS/FAIL/보류 |
| TESTED_DT | 테스트 수행 일시 | |
| TESTER_TEAM | 테스트 담당 부서 | |
| NOTE | 비고 | |

## 배포관리와의 연결
- 테스트 1건은 정확히 1개의 배포를 검증하고, 배포 1건은 여러 차례 테스트를 받을 수 있어 **N:1** 관계입니다.
- **중간 테이블 없이 `DEPLOY_ID` FK로 직접 연결**합니다.

## 연계관리와의 관계
- 참고 그림의 "TST → INT (검증&테스트 → External Connections)" 흐름대로, [14_연계관리](../14_연계관리/README.md) INTERFACE.csv가 `VALIDATED_BY_TEST_ID`로 이 테이블을 참조합니다.

## 생성된 테이블 파일
| 파일 위치 | 설명 | 건수 |
|---|---|---|
| `TEST_CASE.csv` (이 폴더) | 위 필드 정의를 구현한 테스트 마스터 테이블 | 0건(스키마만 우선 반영) |

## 참고 및 관리 방법
- (작성 예정) 테스트 유형별 커버리지 기준, 재테스트 승인 절차
